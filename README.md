# jules-telegram-cockpit

Jules Telegram Cockpit is an enterprise-grade Telegram bot interface for the Jules autonomous AI system. It is built with Node.js, TypeScript, Cloudflare Workers, Hono, and Cloudflare D1.

## Features

- **Zero-Trust Security**: Only explicitly whitelisted users can interact with the bot. Unauthorized users are ignored and automatically kicked from the supergroup.
- **Isolated Sessions**: 1 Repository = 1 Telegram Topic.
- **Jules Integration**: Full bidirectional communication with Jules API (`https://jules.googleapis.com/v1alpha/`).
- **Plan Approval UI**: Inline keyboards for approving or rejecting plans generated by Jules.

## Environment Variables

The application requires the following environment variables (configured via Cloudflare secrets):

- `TELEGRAM_BOT_TOKEN`: The API token for your Telegram Bot.
- `TELEGRAM_SUPERGROUP_ID`: The ID of the Telegram Supergroup where the bot operates.
- `JULES_API_KEY`: The API key to authenticate with the Jules API.

## Telegram Commands

The following commands are available within repository topics:

- `/new [prompt]` - Starts a new active session for the repository.
- `/close` - Archives the currently active session.
- `/list` - Lists up to 10 recent sessions for the current topic.
- `/open <session_id>` - Archives the current active session and sets the specified session as active.

Normal chat messages sent in the topic (when an active session exists) are forwarded to Jules.

## Setup and Deployment

1. Install dependencies:
   ```bash
   npm install
   ```

2. Initialize the database schema:
   ```bash
   npx wrangler d1 execute DB --local --file=./schema.sql
   ```

3. Configure your `wrangler.toml` with the correct D1 Database ID.

4. Deploy to Cloudflare Workers:
   ```bash
   npm run deploy
   ```

## Database Seeding

To populate the `users_whitelist` table with initial administrators, edit the `scripts/seed_admins.sql` file replacing the placeholder values with actual Telegram IDs.

Then run the script against your D1 database:

### Local Development
```bash
npx wrangler d1 execute DB --local --file=./scripts/seed_admins.sql
```

### Production
```bash
npx wrangler d1 execute DB --remote --file=./scripts/seed_admins.sql
```
